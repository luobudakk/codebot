# 代码安全核心（始终启用）

## 工作流概览

1. **定范围**：入口、资产（密钥/PII）、信任边界。
2. **收集证据**：读代码 / 配置文件 / 依赖清单；在条件允许时跑静态工具（见 `static-analysis`、`secrets-and-supply-chain`）。
3. **归类与评级**：每条发现对应类别与严重级别。
4. **可行动修复**：具体改法 + 如何验证（测试、复查命令）。

## 严重级别（建议）

| 级别 | 含义 |
|------|------|
| Critical | 可直接导致远程代码执行、大规模数据泄露或绕过认证且易利用 |
| High | 明显可利用，影响机密性或完整性 |
| Medium | 需特定条件或被利用链组合 |
| Low | 防御纵深、配置硬化、信息泄露边缘 |
| Info | 规范、可维护性、未来风险点 |

## 报告模板（每条发现）

```markdown
### [级别] 标题（简短）

- **位置**: `path:line` 或代码片段引用
- **类别**: CWE-XXX / OWASP API/SRS Top 10（若适用）
- **描述**: 问题是什么
- **影响**: 攻击者能做什么（有假设则写明）
- **修复建议**: 具体步骤或示例
- **验证**: 如何确认已修复（测试、命令、复查点）
```

## CWE / OWASP 使用说明

- 能确定时再写 CWE id；不确定写「疑似与 CWE-XXX 相关」。
- Web/API 可对照 OWASP ASVS / API Security Top 10 的条目名称，无需强行编号。

## 无漏洞时的输出

简要说明**审查了哪些路径/文件**、**未覆盖的范围**，并给出可加强的监控或测试建议（避免空洞的「很安全」）。
